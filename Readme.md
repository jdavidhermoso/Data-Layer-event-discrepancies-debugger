## Description

**DataLayer Discrepancies Debugger** is a browser extension for Chrome that helps developers, analysts, and marketing teams **detect inconsistencies in events sent to the `dataLayer`**.

It is especially useful when working with implementations using tools like Google Tag Manager or analytics platforms such as Google Analytics 4.

The extension listens for `dataLayer.push()` calls on a website and compares events across different pages to identify **property discrepancies**, helping ensure **data consistency across the entire site**, particularly in complex e-commerce implementations.

---

## How It Works

The extension:

1. Intercepts `window.dataLayer.push()` calls.
2. Filters events based on a **configurable event name**.
3. Compares each event with:
   - A **reference event model**.
   - Previously captured events.
4. Detects **property differences**.
5. Displays them in a **floating debug panel**.
6. Allows exporting discrepancies as a **structured JSON report**.

---

## Configuration

The extension is configurable via the **Options page**.

### Target Event

The name of the event that should be monitored.

Example:
```
add_to_cart
purchase
view_item
```

Only events matching this name will be analyzed.

### Model Event

Defines the **reference event structure** used to compare properties.

Example: `add_to_cart`


### Allowed Domains

You can restrict where the extension runs by specifying domains (comma separated).

Example: `example.com, shop.example.com`


The extension will only activate on those domains and their subdomains.

---

## Features

### DataLayer Interception

The extension overrides `window.dataLayer.push()` to capture events in real time.

Captured events are safely serialized to avoid issues with:

- Functions
- DOM elements
- Circular references
- Window objects

---

### Event Comparison

Each captured event is compared against:

#### Reference Model

Ensures the event matches the expected structure.

#### Previous Events

Detects inconsistencies between different pages of the website.

Some keys can be ignored during comparison, such as:
```
timestamp
sessionId
```


---

### Floating Debug Panel

A floating panel appears at the bottom of the page displaying captured events.

Each event shows: `index - page URL - timestamp`


If discrepancies are detected, they are highlighted.

The panel also provides two actions:

- **Clear**: removes all captured events.
- **Export**: generates a JSON with only discrepant properties, grouped by property and URL.

---

## Export Format

Exported discrepancies are grouped by **property** and **URL**.

Example:

```json
{
  "_metadata": {
    "message": "Generated by DataLayer Discrepancies Debugger",
    "linkedin": "https://www.linkedin.com/in/jdavidhermoso/",
    "github": "https://github.com/jdavidhermoso/Data-Layer-event-discrepancies-debugger"
  },
  "price": {
    "/product-a": { "value": 29.99 },
    "/product-b": { "value": 30.00 }
  },
  "currency": {
    "/product-a": { "value": "USD" }
  }
}
```

## Use Cases

- **Analytics QA**: Validate event consistency before releasing analytics implementations.
- **E-commerce Tracking Audits**: Ensure all product events send the same structure across the entire site.
- **Google Tag Manager Debugging**: Quickly detect inconsistent dataLayer values.
- **Marketing and Data Teams**: Confirm that tracking implementations follow the expected schema.

---

## Permissions

- **storage**: to save captured events and configuration settings.
- **host_permissions `<all_urls>`**: allows the extension to run on websites where debugging is needed. Execution is restricted by the Allowed Domains configuration.

## Installation (Development)

1. Download or clone this repository.
2. Open Chrome and go to: `chrome://extensions`.
3. Enable **Developer Mode**.
4. Click **Load unpacked**.
5. Select the project folder.
